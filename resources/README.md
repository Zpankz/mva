# Resources

This directory contains the detailed, explanatory documentation for the MVA heuristic methodology and its supporting document system.

All files in this directory follow `lat.md` conventions.

---

## Purpose

The `resources/` directory holds the "why" and "how" explanations that support the three primary production files at the root:

- `AGENTS.md`
- `LAT.md`
- `SKILL.md`

It also provides guidance on using the templates located in `templates/`.

---

## Recommended Reading Order

1. Start with `resources/system-overview.md` for a high-level understanding.
2. Read `resources/document-lifecycle.md` to understand the flow of MVA → DESIGN + SPEC → GOAL.
3. Consult `resources/using-the-heuristic.md` when preparing to apply the methodology to a new problem.
4. Use the individual template files in `templates/` as the working artifacts.

---

## Key Principles

- `MVA.md` is always a living, iteratively written journal during the discovery phase.
- `DESIGN.md` and `SPEC.md` are created only after a MVA has been validated.
- `GOAL.md` is synthesized from a completed MVA plus its formal technical descendants.
- All documents prioritize agent navigability through `lat.md` conventions.

---

*These resources exist to make the methodology teachable, maintainable, and consistently applied across different problems and practitioners.*
